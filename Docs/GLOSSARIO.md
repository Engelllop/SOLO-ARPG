# Glosario — EMBERVEIL

Términos y siglas usados en documentación y código. Para rutas del repo, ver [`ONBOARDING.md`](ONBOARDING.md).

| Término | Significado |
|---------|-------------|
| **GAS** | *Gameplay Ability System* — habilidades, efectos y atributos (`GameplayAbilities`). |
| **GA** | *Gameplay Ability* — habilidad concreta (ataque, esquivar, etc.). |
| **GE** | *Gameplay Effect* — modificación de atributos o estados en el tiempo. |
| **GDD** | *Game Design Document* — diseño de producto en `Design/GDD/`. |
| **EOS** | *Epic Online Services* — backend/social de Epic; ver `Docs/LEGAL_NOTICES.md`. |
| **UE / UE5** | Unreal Engine 5.x; este proyecto apunta a **5.7** según `EMBERVEIL.uproject`. |
| **UMG** | *Unreal Motion Graphics* — UI declarativa; widgets en C++ con prefijo **`WG_`** en este proyecto. |
| **Data Asset** | Activo de datos en UE para configuración tipada (recetas, tablas de loot, etc.). |
| **FASE 0 / 1 / 2** | Prioridad de producto; ver [`ROADMAP_FASES.md`](ROADMAP_FASES.md). |
| **EMBERVEILCore** | Módulo C++ principal de gameplay. |
| **EMBERVEIL** | Módulo C++ de producto que depende del Core. |
| **PCG** | *Procedural Content Generation* — plugin habilitado en el proyecto. |
| **EOS / OSS** | *Online Subsystem* — abstracción de red; `OnlineSubsystemEOS` en `.uproject`. |

Si añadís términos de dominio (razas, facciones, nombres internos de sistemas), hacedlo en
**una línea** cada uno para mantener el archivo escaneable.
